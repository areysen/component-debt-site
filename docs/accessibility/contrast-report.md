# Accessibility and contrast verification

This is the recorded, defensible artifact for the site's WCAG AA compliance. It
is regenerated by the checks under `scripts/`, which run on every `npm test`. Do
not change a token color in `src/styles/global.css` without re-running the
contrast check and updating this file.

Standard: WCAG 2.1 AA. Thresholds: 4.5:1 for normal text, 3:1 for large text and
non-text UI components. Ratios are computed mathematically (sRGB relative
luminance), not eyeballed. Source: `scripts/check-contrast.mjs`.

Gradient note: the hero, cover, and end-CTA use
`linear-gradient(135deg, #123540, #17576b)`. The lighter stop (`#17576b`) is the
worst case for white text, so white and the light on-gradient text colors are
checked against `#17576b`, per the source assets' stated reasoning for choosing a
dark gradient.

## State is never conveyed by color alone

Verified by forcing the rendered pages to grayscale (see
`docs/screenshots/sample-report-grayscale.png`):

- **Severity** carries a shape glyph and a text label, not just color. From the
  rendered DOM, the `.sev::before` glyphs are triangle `▲` (High), diamond `◆`
  (Medium), and circle `●` (Low), each paired with its word. The finding cards'
  colored left border is a redundant cue only; severity stays legible in
  grayscale.
- **Active navigation** uses `aria-current="page"` plus a bold weight and an
  underline (a non-color cue), not color alone.
- **Good-fit / not-a-fit** columns on `/audit` are labeled in text ("A good
  fit" / "Not a fit"); the column heading colors are redundant.

## Keyboard and focus

- Every interactive element has a visible focus ring:
  `:focus-visible { outline: 3px solid var(--accent2); outline-offset: 2px; }`.
  The ring clears AA as a UI component (6.31:1 on card, 5.86:1 on background).
- A "Skip to content" link is the first focusable element and targets `#main`.
- Tab order on the landing page reaches: skip link, brand, the three nav links,
  the Newsletter outbound link, then the page CTAs, in logical order.

## Images and labels

- No content `<img>` tags ship; the only image is the SVG favicon. There are no
  missing-alt gaps.
- Decorative elements (the numbered pillar markers) are marked
  `aria-hidden="true"` so screen readers do not announce them.
- `<html lang="en">` is set on every page.

## How to reproduce

```
npm test
```

runs `astro check`, the production build, and the contrast, prose, and link
guards. The contrast guard prints this table and exits non-zero if any pair
falls below its threshold.
