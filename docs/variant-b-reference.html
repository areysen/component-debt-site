No design tokens; color and spacing are hard-coded everywhere

Found across 40+ component and stylesheet files

Brand colors, spacing, and font sizes are written as literal values scattered through the codebase rather than referenced from a central source. The same blue appears as three different values, which is three "brand blues" that are meant to be one.

/* card.css */
.card-title { color: #2f6fed; padding: 16px; }
/* banner.css, same intent, different values */
.banner h2  { color: #3170ee; padding: 15px; }
Impact: with a three-layer token system, a brand change becomes a one-line edit instead of a find-and-replace across the codebase, and theming becomes possible at all.

Severity is marked by shape (triangle, diamond, circle) and label as well as color, so it reads for everyone, including colorblind readers.
